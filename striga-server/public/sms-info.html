📱 SMS Verification in Striga Sandbox

How SMS Verification Works

  1. Automatic SMS: When you create a user, Striga automatically sends an SMS verification code
  2. Phone Number Changes: Striga may convert your phone number to Estonian format (+372)
  3. Verification Window: You have 5 minutes to enter the verification code
  4. Resend Option: You can resend the SMS using the /user/resend-sms endpoint

⚠️ Known Issues

  • Phone Number Format: If you enter +49 15257417081, it might be stored as +372 59899637
  • SMS Delivery: In sandbox mode, SMS delivery might be simulated or delayed
  • Test Code: Try using 99999 as the verification code in sandbox mode

✅ Debugging Tips

  • Check the server logs to see the actual phone number stored
  • Try both your original number and the Estonian number for SMS
  • Use the dashboard to monitor user verification status
  • The mobileVerification.dateExpires field shows when the code expires

API Endpoints

Endpoint Purpose Method
/user/create Creates user & sends SMS automatically POST
/user/verify-mobile Verifies the SMS code POST
/user/resend-sms Resends SMS verification code POST
